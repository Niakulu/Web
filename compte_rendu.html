Le projet

Ce site web a été réalisé dans le cadre d'un projet étudiant. Les consignes était de réaliser un site internet marchand en utilisant uniquement HTML et CSS

Les contraintes étaient les suivantes:

  • une page d'accueil avec message du jour, promotions et fonction de recherche,
  • une page de résultat de recherche avec une liste de produits,
  • une page de description détaillée pour un produit sélectionné dans la liste,
  • des éléments de gestion du panier : possibilité d'ajouter un article, voir le contenu du panier, retirer un article du panier,effectuer l'achat,
  • une page "qui sommes-nous ?" de présentation du site, avec le nom des auteurs,
  • une page "détails techniques" avec une description des méthodes employées (Compte-Rendu).

Afin de traviller de concert, nous avons utilisé la plateforme GitHub et avons codé avec Visual Studio Code.

La réalisation

Architecture: HTML

Une fois le sujet du site choisi, la première étape a été de déterminer l'architecture du site. Nous avons déterminé l'ordre des pages et leurs liens entre elles.
Le squelette du site ayant été posé, nous avons commencé par créer la page panier vide et la page panier plein. En effet, ne pouvant utiliser autre chose que html et css, ces deux pages servent à donner l'illusion d'un panier. Ces deux pages ont été codés en utilisant un formulaire et des tableaux.
Nous nous sommes ensuite occupé de la fonction de recherche et de la page de résultats associée. La encore, la recherche est factice. Quelque soit les mots entrés dans la barre (réalisée avec un formulaire et un bouton), l'utilisateur est forcément renvoyé vers une unique page de résultats. Afin d'assurer la cohérence avec le panier factice, seul un article peut être ajouté au panier.
La page de résultats, tout comme la page des produits, contient des photos et du contenus descrivant les catus vendus. Vu le délai imparti, nous n'avons pas eu le temps de coder tout le pages de tous les produits. Nous avons choisi de réaliser seulement une page particulière, accessible en ckiquant sur l'image ou sur le bouton "En savoir plus". Le bouton "En savoir plus" a été rendu inutilisable et la disponibilté modifiée via le CSS afin de justifier l'absence des ces pages.
la page concernant le cactus nain est assez simlple et contient une simple image et une description plus poussée du produit ainsi que la possibilité d'ajout au panier.
La page d'accueil devait contenir le message du jour, la promotion et la description du site. Cette pag est toujours accessibnle en cliquant sur l'image du header.
Les pages concernant l 'équipe et ce compte rendu sont très simple et ne contiennent que du texte mis en forme au moyen de titres.
Toutes les pages sont construites sur le meme modèle : un header, une barre de navigation, un main divisé en section et un footer. De ce fait les différentes pages sont cohérentes avec leur contenu.

Apparence: CSS

Le CSS nous a permis de mettre en forme notre site. Nous avons décidé de mettre une photo de fond pour le header et le footer afin de rendre le site plus attractif (images libres de droit). Nous avons également fait le choix d'utiliser une police inhabituelle qui se charge automatiquement au lancement de la page (police gratuite via Google font).
Le CSS nous apermis de mettre en forme la barre de navigation et les contenus. Nous avons en effet utilisé les propriétés de types Flex.
En utilisant les classes et id, nous avons pu mettre en forme les produits et créer l'illusion d'indisponibilté.
Enfin, le css assure le responsive design de notre site.
